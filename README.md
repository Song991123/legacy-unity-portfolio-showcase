## ✂️ 골라낸 기준

처음에는 6개 프로젝트를 모두 같은 비중으로 배치했지만, 포트폴리오 첫인상 기준으로는 중복과 약한 자료가 섞여 보였습니다. 그래서 결과 화면이 명확하고 면접에서 구현 맥락을 설명하기 쉬운 3개만 대표 사례로 남겼습니다.

## 🧩 I-AI

![I-AI preview](docs/assets/i-ai-screen-2.jpg)

| 항목 | 내용 |
| --- | --- |
| 판단 | 포트폴리오 메인으로 유지 |
| 작업 시기 | 2020 |
| 엔진 | Unity 2019.3.12f1 |
| 포폴 판단 근거 | AI/음성/영상/3D 캐릭터를 한 화면에서 설명할 수 있고, 결과 화면도 가장 직관적입니다. |

매장 안내형 인터랙션을 목표로 음성, 영상, 객체 인식, 3D 캐릭터 표현을 결합해 본 Unity 프로토타입입니다.

- Unity 씬과 캐릭터를 중심으로 서비스 안내 흐름 구성
- OpenCV/YOLO/STT 관련 자료를 결합한 멀티모달 실험
- 캡스톤/해커톤 성격의 프로젝트라 면접에서 구현 맥락을 설명하기 좋음

## 🧩 VI-HO

![VI-HO preview](docs/assets/vi-ho-screen-1.jpg)

| 항목 | 내용 |
| --- | --- |
| 판단 | 포트폴리오 메인으로 유지 |
| 작업 시기 | 2020 |
| 엔진 | Unity 2019.3.12f1 |
| 포폴 판단 근거 | 병실 공간, 캐릭터, 카메라/포즈 인식 실험이 분명해서 I-AI와 다른 축의 Unity 경험으로 보입니다. |

병실형 3D 공간과 캐릭터, 영상 입력/포즈 추정 자료를 결합해 체험형 혼합현실 흐름을 실험한 프로젝트입니다.

- 병실 공간과 캐릭터를 이용한 체험형 시나리오 구성
- OpenCVForUnity, Barracuda/pose 계열 자료 기반 프로토타입
- 시연 화면이 명확해 결과물 중심 포트폴리오 카드로 활용 가능

## 🧩 Recall

![Recall preview](docs/assets/recall-screen-1.jpg)

| 항목 | 내용 |
| --- | --- |
| 판단 | 작게 유지 |
| 작업 시기 | 2019 |
| 엔진 | Unity 2018.2.16f1 |
| 포폴 판단 근거 | 완성도보다는 실제 장비 착용/조작 장면이 강점입니다. 세 번째 카드로만 두면 포트폴리오 폭을 보여줍니다. |

VR 장비와 손/센서 입력을 활용한 체험형 Unity 데모입니다. 실제 착용 시연 자료가 남아 있어 상호작용 경험을 보여주기 좋습니다.

- 실제 착용/조작 장면이 포함된 시연 자료 보존
- 구형 장비 의존 프로젝트의 복원 조건 문서화
- I-AI/VI-HO와 달리 물리 장비 기반 경험을 보여주는 보조 사례

## ✅ 확인한 것

- GitHub Pages에서 실제 페이지가 열리는지 확인
- 대표 카드 3개와 이미지 3장이 정상 로드되는지 확인
- 원본 Unity 소스, 외부 에셋, SDK, 빌드 파일은 공개 repo에 포함하지 않음
- 로컬 경로와 private repo 링크를 공개 문서에서 제거

## ⚠️ 한계

이 저장소는 오래된 Unity 프로젝트의 공개 쇼케이스입니다. 최신 Unity에서 바로 실행되는 제품 저장소가 아니며, 원본 소스와 외부 에셋은 재배포하지 않습니다.
